# 포인터 - 1

C언어에 입문한 사람들이 가장 처음 좌절하는 구간, 포인터에 대해서 다시 공부하였다.

## 포인터란?
- 메모리 주소를 저장하는 변수

말 그대로, 변수가 저장된 메모리(저장공간)의 위치 주소를 가지고 있는 변수다.

### 사용 이유
- 함수에서 원본 수정
    - 주소값을 넘겨서 원본 변수를 바꿀 수 있게 한다.
- 효율적인 데이터 전달
    - 구조체 등 큰 데이터를 통째로 복사하기보다, 주소만 전달함으로서 데이터 양을 크게 줄일 수 있다
- 동적 메모리 관리
    - malloc으로 런타임에 메모리를 할당해놓고 접근은 포인터로 한다.
- 배열, 문자열 처리
    - 배열이나 문자열은 주소가 순서대로 배치되어 있기에 포인터로 연결점이 있다.

## 주소와 포인터

### 주소 연산자 `&`
변수를 선언하면 눈에 보이는 게 다가 아니라, 실제로 그 변수의 데이터는 메모리 어딘가에 저장된다. 그 주소를 알고 싶으면 변수 앞에 `&`연산자를 붙이면 주소를 알 수 있다.

``` C
#include <stdio.h>
{
    int main(void)
    {
        int x = 10;
        printf("x의 값 : %d\n", x);
        printf("x의 주소 : %p\n", &x);

        return 0;
    }
}
```
상단 내용을 실행해보면, 다음과 같다.
```
x의 값 : 10
x의 주소 : (16진수로 이루어진 주소값)
```

여기서 x의 값은 10진수 정수형 형식 지정자(%d)를 사용하는데, x의 주소값은 %p라는 형식 지정자로 되어있다.
이 형식 지정자는 **포인터 형식 지정자**이다.

~~주소값은 16진수로 이루어져 있기에 10진수 정수형으로 받으면 이상하게 출력되거나 오류나는게 당연한데 이건 따로 설명을 안 하길래 아쉬웠다~~

어쨌튼, 정수 형식 지정자에 주소값을 넣으면 이상해지기에, 포인터 형식 지정자에 넣듯이, 이 주소값을 저장하는 곳도 포인터 변수에 넣어야 한다.

```C
...
int *p = &x;
...
```

### 역참조 연산자 `*`
포인터 변수를 선언할 때 *을 변수명 앞에 붙임으로써, 이 변수는 포인터 변수라고 선언된다.

```
x = 일반적이게 사용하는 변수
&x = 변수의 메모리 주소값
p = 포인터 변수(주소값)
*p = 포인터 변수가 가리키는 주소값이 가지고 있는 값(=x의 값)
```

선언할 때는 다음과 같다.

``` C
int a = 10;
int *pointerA = &a;

int *pointerB; //이러면 쓰레기값이 들어가서 역참조해서 프로그램이 죽을 수 있다!

int *pointerB = NULL; // 초기화를 꼭 잊지 말자. 초기화 할 값이 아직 없으면 NULL로 초기화하자.
```
## 포인터로 원본 수정
포인터는 주소를 가리키고 있기에, 원본 데이터를 수정할 수 있다.

``` C
#include <stdio.h>

int main()
{
	int x = 20;
	int *pointer = &x;

	printf("포인터 : %p\n", pointer);
	//16진수로 된 값이 나옴
	printf("포인터의 값 : %d\n", *pointer);
	//pointer이 가리키는 주소의 값이 나옴

	x = 40;
	printf("x의 값 변경 후 *pointer의 값 보기 : %d\n", *pointer);

	*pointer = 50;
	printf("*pointer로 값을 변경한 후 x의 값 : %d\n", x);

	return 0;
}
```

## 포인터와 배열
포인터 변수를 배열에 적용하면 해당 배열의 **첫 번째 원소의 주소**로 변환된다.

``` C
int arr[] = {1, 2, 3, 4, 5};

int *p = arr; //이러면 &arr[0]과 같음. 상세한 주소를 적고 싶으면 &arr[n]을 쓰자.

printf("%d\n", *p);       // 1출력, arr[0] 부분
printf("%d\n", *(p + 1)); // 2출력,  — arr[1] 부분
printf("%d\n", *(p + 2)); // 3출력,  — arr[2] 부분
```
