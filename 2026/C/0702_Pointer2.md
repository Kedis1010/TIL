## 포인터와 배열
배열 이름은 표현식에서 **첫 번째 원소의 주소(&arr[0])**로 변환되고,
포인터에 배열을 넣으면 배열의 첫 번째 원소의 주소값이 저장된다.

``` C
int arr[] = {1, 2, 3, 4, 5};

int *p = arr; //이러면 &arr[0]과 같음. 상세한 주소를 적고 싶으면 &arr[n]을 쓰자.

printf("%d\n", *p);       // 1출력, arr[0] 부분
printf("%d\n", *(p + 1)); // 2출력,  — arr[1] 부분
printf("%d\n", *(p + 2)); // 3출력,  — arr[2] 부분
```

배열 이름은 포인터랑 흡사하게 동작하는 경우가 많아 혼동하는 경우가 있는데, 두 가지는 아예 다른 개념이다.

``` C
int arr[5] = {10, 20, 30, 40, 50};
int *arr_ptr = arr;
```
이렇게 선언된 배열 arr와 그 배열의 주소를 가리키는 포인터 변수 arr_ptr이 있다고 가정하자.

``` C
printf("%d\n", arr[2]);        // 30
printf("%d\n", arr_ptr[2]);    // 30
printf("%d\n", *(arr+2));      // 30
printf("%d\n", *(arr_ptr+2));  // 30
```

상단 내용처럼 입력할 경우, 포인터 변수 arr_ptr과, 배열 arr의 값을 확인할 때 결과값이 동일하게 출력된다.

하지만, 구조적으로 뜯어보면 다른 걸 알 수 있다.

``` C
sizeof(arr);
// 결과값 20
// int형(4바이트) 변수가 5개
sizeof(arr_ptr);
// 결과값 8
// 주소값을 저장하는 포인터 변수 자체의 크기(8바이트)를 의미하기에 다르다.

arr = arr_ptr;      
// ❌ 컴파일 에러가 남.
// 배열에 주소를 쑤셔넣으려는 것.
arr_ptr = arr;      
// ✅ 정상 실행
// arr배열의 주소를 포인터 변수에 저장하는 행위.

arr++;
// 배열 이름으로는 다음 항목으로 이동이 불가능하다.
arr_ptr++;          
// 배열을 주소값으로 보면, 순차적으로 나열되어 있기 때문에 주소값을 의미하는 포인터 변수의 값을 1 증가시키면 타입의 크기(이 경우 int형 4바이트만큼)만큼 이동하여 다음 항목으로 이동됨.
```

배열 이름은 **첫 번째 원소의 주소로 변환될 수 있는 이름**이며, 주소를 저장할 수 있는 **변수가 아니다**. 
따라서 **포인터 변수처럼 다른 주소를 저장하거나 변경할 수 없다**.

## 포인터의 헷갈리는 점들

보통 포인터 입문하는 사람들은 값을 복사하는 것과 주소 공유하는 것을 구분하기 힘든 경우가 많은데, 그냥 간단히 구조만 파악하며 읽어보면 쉽게 알 수 있다.

``` C
int score = 10;
int *score_ptr = NULL;
int backup;

// 1. 변수의 값 복사
backup = score; //score에 있는 값을 backup으로 복사함.
backup = 99; //99로 값을 변경함.
printf("score : %d, backup : %d", score, backup);
// 출력 결과값 : score : 10, backup : 99

// 2. 주소값 공유
score_ptr = &score; //포인터는 선언할 때와 역참조할 때만 *을 쓴다. 이미 선언을 상단에 해 두었고, 역참조가 아니라 주소를 저장할 것이기에 *기호는 필요 없음.
*score_ptr = 99; //역참조(*)로 주소값에 있는 값을 변경함.

printf("*score_ptr : %d, score : %d", *score_ptr, score);
// 출력 결과값 : *score_ptr : 99, score : 99
// 원본 주소를 역참조해서 원본 값이 바뀐다
```

그리고 포인터 변수 여러 개가 같은 주소를 가리켜도, **사실상 전부 같은 변수 하나를 가리키는 거나 다름없기에** 어떤 곳에서 접근해서 값을 바꾸던, **조회 시 전부 값이 갱신되어 동일하게 보인다**.
~~뭔가 말이 이상한데, 그냥 다 같은 변수를 가리키니까 당연한 말이다~~

``` C
int x = 10;
int *first  = &x;
int *second = &x;
int *third  = first;

*second = 50;
printf("%d %d %d %d\n", x, *first, *second, *third);

//결과값은 전부 50이다.
```

## 포인터의 포인터

포인터 변수도 '주소값을 저장하는 **변수**'이기에 그 포인터 변수도 주소값이 존재한다.
그래서, 포인터 변수의 주소값을 포인터 변수로 저장할 수 있다 ~~**마트료시카**~~

``` C
int a = 10;
int *a_ptr = &a;  
int **a_ptr_ptr = &a_ptr; 
```
이렇게 하면 a_ptr은 a의 주소값을 가지고 있고, a_ptr_ptr은 a_ptr이라는 포인터 변수의 주소값을 가지게 되고, *a_ptr_ptr은 a_ptr의 값(a의 주소값)을 가지게 되고, **a_ptr_ptr은 역참조를 두 번 하여 a의 값을 가지게 된다.

함수에서 포인터 자체를 바꿀 때 사용한다.

``` C
void change_value(int *target) {
    *target = 999;
}

void change_target(int **target_ptr, int *new_addr) {
    *target_ptr = new_addr;
}

int main(void) {
    int a = 10, b = 20;
    int *current = &a;

    printf("%d\n", *current);       // 10 — a를 가리킴

    change_value(current);
    printf("%d\n", a);              // 999 — a의 값이 바뀜

    change_target(&current, &b);
    printf("%d\n", *current);       // 20 — 이제 b를 가리킴

    return 0;
}
```
